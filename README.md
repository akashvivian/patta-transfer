# Patta Land Ownership Transfer Web Application

A beginner-friendly, internship-level full-stack web application designed to digitize and streamline the land title ownership transfer (Patta Transfer) process. Built using a layered architecture in **Java Spring Boot** (without Thymeleaf template coupling) as a stateless REST API, and a highly responsive, modern glassmorphic frontend utilizing **HTML, CSS, Bootstrap 5, and JavaScript Fetch API**.

---

## 🏗️ Project Architecture

This application utilizes a decoupled, separate frontend and backend architecture:

```
[ Citizen/Admin Frontend ]
      │ (HTML/CSS/Bootstrap 5/JS Fetch API)
      ▼
   [ CORS Configuration ] (Allows communication between local files/different origins)
      │
      ▼
[ Spring Boot REST API ] (Listening on localhost:8080)
      │
      ├── [ Controller Layer ] (Exposes REST endpoints, handles Multipart Files)
      ├── [ Service Layer ]    (Houses core business logic and local file storage logic)
      ├── [ Repository Layer ] (Executes DB access via Spring Data JPA & Hibernate)
      ▼
  [ MySQL Database ]    (Stores Users & Land Title Application details)
```

---

## 🛠️ Technology Stack

* **Backend Framework**: Java Spring Boot 3.3.0
* **Data Access Layer**: Spring Data JPA, Hibernate ORM
* **Database**: MySQL 8.x
* **Build Automation**: Maven
* **Frontend Components**: HTML5, Vanilla CSS3, Bootstrap 5.3, FontAwesome Icons (via CDN)
* **API Integration**: JavaScript Promise-based Fetch API

## 🗄️ Database Tables & Schema

The MySQL database contains two primary tables with relational integrity:

### 1. `users` Table
Stores registered applicants and administrative personnel.
* `id` (BIGINT, Primary Key, Auto-Increment)
* `name` (VARCHAR, Full Name of the account holder)
* `mobile` (VARCHAR, Verified contact number)
* `address` (VARCHAR, Postal Address)
* `email` (VARCHAR, Unique email used as username)
* `password` (VARCHAR, Login password)
* `role` (VARCHAR, Account permissions: `'USER'` or `'ADMIN'`)

### 2. `patta_applications` Table
Stores land transfer applications linked to the filing user.
* `id` (BIGINT, Primary Key, Auto-Increment)
* `user_id` (BIGINT, Foreign Key referencing `users(id)` with cascade delete)
* `applicant_name` (VARCHAR, Name of the new proposed owner)
* `survey_number` (VARCHAR, Land plot coordinate number)
* `patta_number` (VARCHAR, Current official title record number)
* `transfer_type` (VARCHAR, Category: `'ISD'` or `'NISD'`)
* `document_type` (VARCHAR, Registered Deed e.g. `'Sale Deed'`, etc.)
* `property_document` (VARCHAR, Name of property document saved on disk)
* `identity_proof` (VARCHAR, Name of identity document saved on disk)
* `address_proof` (VARCHAR, Name of address document saved on disk)
* `status` (VARCHAR, Application review stage: `'Pending'`, `'Approved'`, `'Rejected'`)

## 🚀 Step-by-Step Setup Guide

Follow these simple instructions to set up and run the application locally on your computer:

### Prerequisites
* **Java Development Kit (JDK)**: Version 17 or higher installed.
* **Apache Maven**: Installed and added to environment variables PATH (or use `mvnw` wrapper).
* **MySQL Server**: Running locally on default port `3306`.

---

### Step 1: Initialize the MySQL Database
1. Open your MySQL Command Line Client, MySQL WorkBench, or phpMyAdmin.
2. Log in using your root credentials.
3. Run the following command to create the database:
   ```sql
   CREATE DATABASE IF NOT EXISTS patta_transfer;
   ```
   *(Note: The database tables will be automatically generated by Hibernate JPA on backend startup! If you wish to review the exact table definitions manually, check the [schema.sql](file:///e:/nic/backend/src/main/resources/schema.sql) file).*

---

### Step 2: Configure Application Database Credentials
1. Navigate to the backend properties file: `backend/src/main/resources/application.properties`.
2. Locate the database username and password properties:
   ```properties
   spring.datasource.username=root
   spring.datasource.password=root
   ```
3. Update `root` and `root` to match your local MySQL server credentials.

---

### Step 3: Run the Spring Boot Backend REST API
1. Open a terminal/command prompt window and navigate to the backend folder:
   ```bash
   cd backend
   ```
2. Build and run the project using Maven:
   ```bash
   mvn spring-boot:run
   ```
3. Once running, you should see console logs indicating:
   * Database tables verified/updated.
   * A local file storage directory named `uploads` created automatically in the backend directory.
   * Default administrator seeded successfully:
     * **Email**: `admin@patta.com`
     * **Password**: `admin123`
     * **Role**: `ADMIN`
   * Backend successfully listening on: **`http://localhost:8080`**

---

### Step 4: Open and Explore the Frontend
Since the frontend and backend are completely decoupled, you do **not** need to compile or run the HTML files through Maven.
1. Navigate to the `frontend/` folder.
2. Double-click the `index.html` file to open it directly in Google Chrome, Microsoft Edge, or Mozilla Firefox.
3. Alternatively, you can serve it via a local static server like **Live Server** extension in Visual Studio Code for a smoother experience.

## 📝 Key Learning Points for Interns
1. **Decoupled Architecture**: Separation of UI and Business layers allows for easy migration (e.g. changing frontend to React or Angular without altering the Java APIs).
2. **State Management**: Simple utilization of standard browser `localStorage` acts as a session token provider, maintaining authentication states without heavy cookies or security frameworks for demo projects.
3. **Multipart Form Processing**: Uploading multiple files and processing them alongside entity fields requires special parsing, which is handled effortlessly in Spring via `MultipartFile`.
4. **Collision Resistance**: Appending a Unix timestamp prefix (`System.currentTimeMillis()`) to uploaded files guarantees no file uploaded will ever overwrite another file with the same name on the server.
5. **CORS Enabling**: Modern browsers block requests to a backend on `localhost:8080` if the webpage is loaded from `file:///` or another origin. Standard global mapping in `@Configuration WebMvcConfigurer` solves this problem.
